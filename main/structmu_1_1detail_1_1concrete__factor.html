Detailed Description

Holds a member for each value defined in mu::detail::factor_traits.

Most members map over to this structure obviously. However, the factor_traits for a factor type T, define a member type base. The concrete factor cannot store a type as a member, but it can store a unique id for that type. The id is assigned by the concrete_factor_generator type.
